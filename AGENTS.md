## 产品主线

当前产品名是 2FA 动态验证器。

2FA 动态验证器是通用 RFC 6238 / TOTP 静态浏览器工具。用户把自己持有的 TOTP 信息导入浏览器，本工具在本地生成当前验证码，并提供账号列表、二维码输出和迁移导入能力。

当前能力：

- 粘贴 `otpauth://totp/...` 链接导入。
- 粘贴裸 Base32 `secret` 导入。
- 粘贴一行一个 `otpauth://` 链接的迁移文本导入。
- 上传二维码图片导入。
- 摄像头弹窗扫码导入。
- 浏览器 localStorage 保存多个 TOTP 账号。
- 当前账号验证码展示、复制和环形倒计时。
- 本地账号选择、删除和清空。
- 当前账号标准二维码生成、链接复制和二维码图片下载。
- GitHub Pages 静态部署。

## 技术纪律

- 应用使用 TypeScript、Vite、原生 DOM、Bootstrap、Bootstrap Icons、OTPAuth、QRCode、@zxing/browser 和 Vitest。
- 运行时代码通过 npm 依赖和构建产物固定，不从 CDN 加载会接触密钥的代码。
- TOTP、otpauth 解析、导入输出序列化和存储规则放在可测试模块中。
- 页面文件负责交互编排和展示，复杂业务规则放入 `src/core/`、`src/storage/`、`src/qr/` 或 `src/ui/` 专门模块。
- `src/main.ts` 负责应用启动入口。
- `src/ui/controller.ts` 负责 DOM 事件、摄像头、复制、下载和删除等外部接线。
- `src/ui/view/` 负责 HTML 模板。
- `src/styles/` 按基础、布局、输入、验证码、账号、辅助面板和响应式拆分。
- 只有当拆分能降低真实复杂度、隔离变化、提升测试性或匹配现有架构边界时，才新增模块或抽象。
- 避免用 `any` 或无类型对象逃避设计。

## 安全纪律

- TOTP `secret` 等同于 2FA 根密钥。
- `otpauth://` 链接、二维码图片和迁移文本都按敏感数据处理。
- 密钥只进入浏览器内存、localStorage 和用户主动复制或下载的本地输出。
- 密钥不得进入日志、测试快照、错误上报、统计、远程配置、URL query 或第三方网络请求。
- 扫码、上传二维码图片、粘贴文本、复制链接和下载二维码都必须保持本地处理边界。
- UI 和文档描述本工具承担的能力时，以本地验证器和迁移辅助工具为准。

## UI 纪律

- 第一屏就是工具本身。
- 输入和输出在左侧同一个工具面板内。
- 右侧展示当前验证码和本地账号。
- 桌面端左右两栏等高；内容增多时在面板内部滚动。
- 复制、删除、错误和扫码反馈使用 toast 或弹窗，不占用主布局高度。
- 摄像头扫码使用弹窗。
- 倒计时使用环形进度，按秒更新并带平滑过渡。
- 当前账号输出为二维码、复制链接和下载图片。
- 移动端按单列布局呈现，按钮、输入框和账号列表保持可操作。

## 工程入口

本地运行：

- `npm.cmd install`：安装依赖。
- `npm.cmd start`：调用 `start.py` 构建 `dist/`，启动静态服务并打开无痕/隐私浏览器。
- `python .\start.py --check`：检查本地启动入口。
- `python .\start.py --no-build --no-open --port 4568`：服务现有 `dist/`。

开发验证：

- `npm.cmd test`：运行 Vitest。
- `npm.cmd run typecheck`：运行 TypeScript 类型检查。
- `npm.cmd run build`：类型检查并构建 `dist/`。

自动部署：

- `.github/workflows/deploy.yml` 在 push 到 `main` 后运行安装、测试、类型检查、构建和 GitHub Pages 部署。

## 验证规则

- 文档任务必须检查目标文件是否存在。
- 代码任务收尾前必须运行项目定义的类型检查、测试、构建或等价验证。
- 生产级收口至少运行 `npm.cmd test`、`npm.cmd run typecheck`、`npm.cmd run build`。
- 修改本地启动入口时运行 `python .\start.py --check`。
- TOTP 正确性必须用 RFC 6238 测试向量保护。
- 导入解析必须覆盖 `otpauth://`、裸 secret、二维码解析失败、无效 secret、HOTP 拒绝。
- 导入输出必须覆盖迁移文本导入、当前账号二维码生成、链接复制来源和敏感字段不丢失。
- 不能只凭解释声称完成，必须说明验证结果。
- commit 或 push 前必须说明将提交的范围并得到 owner 明确确认。
