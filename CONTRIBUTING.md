# Contributing

欢迎提交改进。改动应基于真实问题、当前产品事实和可验证结果。

## 开发流程

1. 先确认问题能复现，或确认需求有明确输入、输出和业务规则。
2. 修改代码时保持范围清晰，相关重构和行为变化放在同一个可解释边界内。
3. 行为、配置或用户路径变化时，同步更新测试和文档。
4. 提交前运行项目定义的相关验证命令。

## 文档

- 产品事实统一写在 `spec.md`。
- 根目录 `README.md` 放项目入口和当前功能概览。
- `AGENTS.md` 写 agent 协作规则和仓库纪律。
- 未实现能力写入计划、未知点或后续任务。

## 代码风格

- 沿用现有目录结构和实现方式。
- 使用简单、可维护、可测试的实现。
- 依赖服务真实复杂度。
- 错误处理给出明确提示。
- 文件职责按变化原因划分。

## 提交前

- 检查工作区，确认只包含本次任务相关文件。
- 运行 `npm.cmd test`、`npm.cmd run typecheck`、`npm.cmd run build`。
- 修改本地启动入口时运行 `python .\start.py --check`。
- 移除真实业务数据、账号、密钥、Token、个人数据、临时本地文件和构建产物。
- commit 和 push 由项目 owner 明确确认后执行。
