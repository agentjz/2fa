# 2FA UI

UI 服务一个核心路径：用户在平台要求 2FA 时，快速选择账号、读取或复制当前验证码。

## 当前信息架构

桌面端：

- 左侧工具面板：输入区和输出区。
- 右侧工作区：当前验证码卡片和本地账号列表。

移动端：

- 单列排列。
- 输入、输出、验证码和账号列表保持可操作。

## 输入区

- 支持粘贴 `otpauth://totp/...` 链接。
- 支持粘贴裸 Base32 `secret`。
- 支持粘贴迁移文本。
- 支持二维码图片上传。
- 支持摄像头弹窗扫码。
- 粘贴有效内容后自动解析、自动保存、自动显示验证码。
- 清空输入按钮放在输入区底部操作行。
- 裸 secret 可填写服务方和账号名。

## 验证码区

- 显示当前账号名。
- 显示大号验证码。
- 复制验证码使用明确按钮。
- 倒计时使用环形进度。
- 环形进度按秒更新，并用 CSS 过渡滑到下一格。
- 算法、位数和周期作为紧凑元信息展示。

## 账号区

- 本地账号列表适合扫描。
- 长账号名不能挤压验证码和按钮。
- 当前账号有明确选中态。
- 账号列表内容增多时在账号面板内部滚动。
- 删除和清空操作使用确认。

## 输出区

- 输出当前账号二维码。
- 提供复制当前账号 `otpauth://` 链接。
- 提供下载当前账号二维码图片。
- 二维码生成中显示稳定占位。
- 输出区和输入区在同一个左侧工具面板内。

## 弹窗和反馈

- 摄像头扫码使用弹窗。
- 弹窗关闭按钮使用圆形图标按钮。
- 复制、删除、扫码和错误反馈使用 toast。
- toast 使用固定浮层，不占用主布局高度。
- 桌面端左右两栏高度保持一致。

## 视觉规则

- 使用 Bootstrap 和 Bootstrap Icons 承载基础组件。
- 自定义 CSS 只做产品气质、布局细节和必要覆盖。
- 卡片圆角保持 8px。
- 页面第一屏就是工具本身。
- 按钮文本和图标不溢出。
- 桌面端避免整页高度跳变。
- 移动端避免横向溢出和难用的嵌套滚动。

## 验证

收尾前检查：

- 首屏能直接导入或查看验证码。
- 输入有效 TOTP 后自动出码。
- 清空输入后当前输入预览回到空态。
- 当前账号二维码、复制链接和下载图片可用。
- 长账号名不挤压布局。
- 本地账号列表在面板内滚动。
- toast 不改变左右栏高度。
- 摄像头弹窗不改变左右栏高度。
- 移动端按钮和输入框可操作。
- 错误提示清晰。
