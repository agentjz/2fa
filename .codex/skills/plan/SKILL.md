# Plan

`plan.md` 是单文件规格驱动执行合同。中大型任务、跨模块重构、产品边界调整、工程实现和验证闭环都使用它承载当前任务事实。

## 写作内容

`plan.md` 覆盖：

- 需求。
- 当前事实。
- 生产级验收保护点。
- 目标。
- 边界。
- 设计。
- 实施任务。
- 验证计划。
- 收口。

## 执行规则

- 写完计划，再动手。
- checklist 随进度更新。
- 新事实推翻计划，先改计划，再继续。
- 判断和实现基于文件、命令、测试、输出或 owner 明确输入。
- research、设计、实现、测试、文档同步、验证和收口形成同一次完整交付。
- 分阶段只表示交付边界，不表示降低标准。
- 当期范围按生产级标准完成。

## 2FA 计划要点

- 需求写用户路径：导入、取码、本地账号、二维码输出、迁移导入、静态部署。
- 当前事实写代码、测试、文档、配置和命令输出。
- 设计写清 `src/core/`、`src/storage/`、`src/qr/`、`src/ui/`、`src/styles/` 和测试边界。
- 安全边界写清 `secret`、`otpauth://`、二维码图片和迁移文本的本地处理路径。
- UI 任务写清桌面等高、内部滚动、toast、摄像头弹窗、环形倒计时和移动端适配。
- 验证计划至少包含 `npm.cmd test`、`npm.cmd run typecheck`、`npm.cmd run build`。

## 收口

收口写清：

- 完成的当前事实。
- 运行的命令。
- 通过的测试。
- 没有验证的内容。
- 剩余风险。
- commit / push 状态。
