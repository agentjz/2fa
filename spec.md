# 2FA 动态验证器 Spec

`spec.md` 是当前产品事实主干。它记录 2FA 动态验证器的产品定位、核心概念、用户路径、功能范围、数据边界和验收标准。

## 1. 产品定位

2FA 动态验证器是通用 RFC 6238 / TOTP 静态浏览器工具。

用户把自己持有的 TOTP 信息导入浏览器，本工具根据当前时间和 TOTP 密钥生成动态验证码。二维码导入、迁移文本导入、本地账号列表、当前账号二维码输出和 GitHub Pages 静态部署构成当前产品主线。

## 2. 当前能力

- 粘贴 `otpauth://totp/...` 链接导入。
- 粘贴裸 Base32 `secret` 导入。
- 粘贴一行一个 `otpauth://` 链接的迁移文本导入。
- 上传二维码图片导入。
- 摄像头弹窗扫码导入。
- 浏览器 localStorage 保存多个 TOTP 账号。
- 当前账号验证码展示和复制。
- 当前账号环形倒计时。
- 本地账号选择、删除和清空。
- 当前账号标准二维码生成。
- 当前账号 `otpauth://` 链接复制。
- 当前账号二维码图片下载。
- GitHub Pages 静态部署。

## 3. 核心概念

### 3.1 TOTP

TOTP 是基于时间的一次性验证码。服务平台和本地验证器持有同一个密钥，双方用当前时间、密钥、算法、位数和周期计算验证码。

当前默认值：

- 默认算法：SHA-1。
- 默认位数：6 位。
- 默认周期：30 秒。
- 兼容算法：SHA-1、SHA-256、SHA-512。

### 3.2 otpauth 链接

标准 TOTP 导入链接形如：

```text
otpauth://totp/<issuer>:<account>?secret=<base32-secret>&issuer=<issuer>&digits=6&period=30&algorithm=SHA1
```

二维码是该链接的图像编码。扫码导入、二维码图片导入、粘贴链接导入和迁移文本导入最终都会进入同一套 TOTP 账号解析流程。

### 3.3 Secret

`secret` 是 TOTP 根密钥。当前工具把 `secret`、`otpauth://` 链接、二维码图片和迁移文本都按敏感数据处理。

### 3.4 本地保存

本地保存使用当前浏览器的 localStorage。保存对象是 TOTP 账号列表，验证码根据当前时间动态计算。

## 4. 用户路径

### 4.1 首次绑定

1. 用户在支持 TOTP 的平台开启 2FA。
2. 平台显示二维码或 setup key。
3. 用户在本工具中扫码、上传二维码图片、粘贴 `otpauth://` 链接，或粘贴裸 `secret`。
4. 本工具解析账号、服务方、密钥、算法、位数和周期。
5. 本工具显示当前验证码和倒计时。
6. 用户把验证码填回平台完成绑定确认。

### 4.2 日常登录

1. 平台要求输入 2FA 验证码。
2. 用户打开本工具。
3. 用户在本地账号列表中选择账号。
4. 用户复制或读取当前验证码。
5. 用户填入平台完成登录。

### 4.3 迁移

1. 用户在本工具中选中账号。
2. 用户复制当前账号 `otpauth://` 链接，或下载当前账号二维码图片。
3. 用户在新浏览器或新验证器中导入该链接或二维码。
4. 导入后生成的验证码与原工具一致。

## 5. 页面结构

桌面端页面由左右两栏组成：

- 左侧工具面板：输入区和输出区。
- 右侧工作区：当前验证码卡片和本地账号列表。

交互规则：

- 输入区粘贴有效内容后自动解析、自动保存、自动显示验证码。
- 清空输入后当前输入预览回到空态。
- 输出区展示当前账号二维码，提供复制链接和下载图片。
- 摄像头扫码使用弹窗承载。
- toast 用于复制、删除、扫码和错误反馈。
- 桌面端左右两栏等高；账号列表内容增多时在账号面板内部滚动。
- 移动端使用单列布局。

## 6. 数据边界

当前数据对象：

- `TotpAccount`：一个 TOTP 账号。
- `issuer`：服务方名称。
- `label`：账号名或显示名。
- `secret`：Base32 TOTP 密钥。
- `algorithm`：HMAC 算法。
- `digits`：验证码位数。
- `period`：验证码周期秒数。

数据位置：

- 账号列表保存到 localStorage。
- 当前验证码只在运行时计算和展示。
- 当前账号二维码和链接由用户主动复制或下载。

## 7. 安全边界

- 密钥只在浏览器本地处理。
- TOTP 计算在浏览器本地完成。
- 二维码识别、二维码生成和迁移文本解析在浏览器本地完成。
- 运行时代码通过 npm 依赖和构建产物固定。
- 密钥不得进入日志、测试快照、错误上报、统计、远程配置、URL query 或第三方网络请求。

## 8. 技术实现

- TypeScript。
- Vite。
- 原生 DOM。
- Bootstrap。
- Bootstrap Icons。
- OTPAuth。
- QRCode。
- @zxing/browser。
- Vitest。
- GitHub Actions。

模块职责：

- `src/core/totp.ts`：TOTP 类型、解析、归一化、验证码生成和 URI 序列化。
- `src/storage/accountsStore.ts`：localStorage 读写、去重和恢复。
- `src/qr/qr.ts`：二维码图片解析、摄像头扫码接线和二维码生成。
- `src/ui/controller.ts`：DOM 事件、摄像头、复制、下载和删除等外部接线。
- `src/ui/autoImport.ts`：输入自动解析、自动保存和输入状态判断。
- `src/ui/state.ts`：应用状态创建和当前账号选择器。
- `src/ui/view/`：页面 HTML 模板。
- `src/styles/`：基础、布局、输入、验证码、账号、辅助面板和响应式样式。

## 9. 验收标准

- 标准 TOTP `otpauth://` 链接可以导入并显示验证码。
- 裸 Base32 `secret` 可以导入并显示验证码。
- 迁移文本可以导入多个账号。
- 二维码图片可以解析 TOTP 账号。
- 摄像头弹窗可以扫码导入 TOTP 账号。
- 多个账号可以保存到当前浏览器。
- 已保存账号可以选择、删除和清空。
- 当前验证码可以复制。
- 倒计时使用环形进度并按秒平滑过渡。
- 当前账号可以生成标准二维码。
- 当前账号 `otpauth://` 链接可以复制。
- 当前账号二维码图片可以下载。
- 无效 secret 有明确错误提示。
- HOTP 链接被拒绝。
- RFC 6238 官方测试向量通过。
- 构建产物可以部署到 GitHub Pages。
- 页面在桌面端和移动端都可用。
