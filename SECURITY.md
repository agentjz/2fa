# Security Policy

## 支持范围

当前维护主分支中的最新代码。

## 报告安全问题

发现安全问题时，通过仓库 Issue 或项目维护者可用的私有渠道联系，并提供：

- 受影响的功能、页面、文件或流程。
- 复现步骤。
- 可能影响的数据或操作。
- 受影响的版本、提交或环境。
- 建议修复方向（如有）。

## 敏感材料处理

提交问题、截图、日志或复现材料前，移除：

- 账号、密码、Token、Cookie、私钥或证书。
- 业务数据、个人数据和内部链接。
- TOTP `secret`、验证码、恢复码、`otpauth://` 链接和二维码截图。
- 本地配置、localStorage 导出和临时调试文件。

## 修复原则

- 安全问题以可复现事实和当前代码边界为准。
- 修复直接处理破坏边界的路径。
- 修复后补充或更新测试，覆盖真实可观察风险。
